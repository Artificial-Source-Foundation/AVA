# Archives

Historical sprint plans, epic designs, and completed work items.
These are retained for reference but are no longer actively maintained.

## Folder Structure

- **`completed-sprints/`** — Sprint execution plans, backlogs, and QA checklists
  - `rust-era/` — Rust-era sprint prompts (current development phase)
  - Contains sprint backlogs, changelogs, integration tracking, and QA checklists
- **`completed-epics/`** — Epic design documents and feature specifications
- **Root files** — General archived documentation (vision, roadmap, design system, reference materials)

## Root Files

| File | Description |

| `VISION.md` | Project vision and design philosophy |

| `reference-repos.md` | Reference repositories for comparison |
| `testing-models.md` | SOTA models for testing documentation |
