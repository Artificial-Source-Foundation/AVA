# Sprint 2.3 Plugin UX Wiring

## Goal

Deliver end-to-end plugin lifecycle actions from the settings UI using real backend extension manager flows.

## Ticket board

- `INT-001` Lifecycle wiring (install/enable/disable/uninstall) - in_progress
  - Owner backlog: `docs/development/backlogs/integration-backlog.md`
  - Owner checklist:
    - [ ] Wire install action in settings UI to backend extension manager call.
    - [ ] Wire enable/disable action in settings UI to backend extension manager call.
    - [ ] Wire uninstall action in settings UI to backend extension manager call.
    - [ ] Verify desktop runtime flow for install, enable/disable, uninstall.
- `INT-002` State sync + optimistic action handling - in_progress
  - Owner backlog: `docs/development/backlogs/integration-backlog.md`
  - Owner checklist:
    - [ ] Define optimistic UI transition states for lifecycle actions.
    - [ ] Reconcile final backend state after success/failure responses.
    - [ ] Prevent stale toggles during in-flight lifecycle requests.
    - [ ] Add test coverage for rapid toggles and retry sequences.
- `INT-003` Failure UX + recovery actions - in_progress
  - Owner backlog: `docs/development/backlogs/integration-backlog.md`
  - Owner checklist:
    - [ ] Map backend lifecycle errors to actionable UI states.
    - [ ] Provide retry action for transient failures.
    - [ ] Provide recover/uninstall action for unrecoverable failures.
    - [ ] Verify failure paths in desktop runtime and smoke tests.
- `S2.3-UI-01` Featured/search/catalog polish - todo
  - Owner backlog: `docs/frontend/backlog.md`
  - Owner checklist:
    - [ ] Finalize featured plugin cards and metadata fields.
    - [ ] Implement search behavior and empty/error states.
    - [ ] Ensure catalog details link to lifecycle controls.
