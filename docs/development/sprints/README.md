# Sprint Execution Docs

Use one file per sprint named `YYYY-<id>-<topic>.md`.

## Active Sprint Docs

- `docs/development/sprints/2026-S1.6-testing-hardening-closeout.md`
- `docs/development/sprints/2026-S2.3-plugin-ux-wiring.md`
- `docs/development/sprints/2026-DX-1-docs-architecture-hardening.md`

Required sections in every sprint doc:

1. Goal
2. Ticket board
3. Dependencies
4. Evidence
5. Exit criteria
6. Close checklist

Rules:

- Keep ticket states current (`todo`, `in_progress`, `done`, `blocked`).
- Link each ticket to the owner backlog (frontend, backend, integration).
- Do not duplicate full backlog details; link to source docs.
- Sprint cannot be marked complete until close checklist is fully checked.
